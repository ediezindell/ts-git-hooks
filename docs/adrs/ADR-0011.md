## ADR-0011: 設定 validation を fail-closed 化

**日時**: 2026-05-03T18:00:00+09:00
**タスク概要**: `core/config.ts:149-155` の valibot 失敗時の挙動を `console.warn` → 続行 (fail-open) から `logger.error` → `return null` (fail-closed) に変更する。`runHook` 側はこの null を「config がロードできなかった」と同等に扱う (既存挙動)。

### コンテキスト

現状:
```ts
const result = v.safeParse(ConfigSchema, configModule.config);
if (!result.success) {
  console.warn(`Invalid configuration in ${_configFileName}:`, ...);
}
_memoizedConfig = normalizeConfig(configModule.config as TSGitHookConfig);
return _memoizedConfig;
```

invalid な config (typo, 非対応キー) でも warn だけ出して通る。例えば `'pre-commit': { replayFormatter: true, config: {...} }` のような silent-no-op 構造 (ADR-0003 で型側は対応済み) は valibot で reject されるが warn 止まり。user は warn を見落とすと「設定したのに動かない」体験を踏む。

### 決定内容

```ts
const result = v.safeParse(ConfigSchema, configModule.config);
if (!result.success) {
  logger.error(
    `Invalid configuration in ${_configFileName}:\n${
      JSON.stringify(v.flatten(result.issues).nested, null, 2)
    }`,
  );
  return null;
}
_memoizedConfig = normalizeConfig(configModule.config as TSGitHookConfig);
return _memoizedConfig;
```

挙動変化:
- invalid config → `loadConfig` が `null` 返す → `runHook` が `Configuration file not found.` を error し `false` を返す → hook 失敗
- 既存の「config 不在時」と同じ路径に乗る

### 決定理由

- silent-no-op (typo した config が知らない間に効いていない) を撲滅
- valibot schema は ADR-0003 で `replayFormatter` per-hook 排除済み、現在の schema は意図的な validation。fail-closed が schema 設計と整合
- `logger.error` でユーザに具体的な issue (どのキーが invalid か) を提示するので debugging 体験が改善

### 却下した選択肢

| 選択肢 | 却下理由 |
|--------|----------|
| warn を強調する (色付け、複数行) | 結局 warn は無視される。runtime の挙動と doc の主張が食い違う問題は解けない |
| `--strict` flag で opt-in fail-closed | flag を知るユーザは既に doc を読めるので恩恵が薄い。default が安全な方が library として正しい |
| invalid 部分だけ削って続行 | 「user が書いた config と動いた config が違う」は最悪パターン。silent-corruption |

### 影響範囲

変更されるファイル:
- `src/core/config.ts` — `console.warn` → `logger.error`、early `return null` 追加
- `src/core/config.test.ts` — 新規テスト: invalid config で `loadConfig` が null を返すこと、error log が出ること

挙動変化:
- 既存ユーザで「warn 出ているが動いていた」 config は hook が失敗するようになる
  - これは silent-no-op を顕在化させる意図された挙動
  - error message に invalid 箇所が出るので修正は容易
- valibot schema の bug (false positive) があれば既存ユーザが詰まる
  - schema は ADR-0003 で見直し済み

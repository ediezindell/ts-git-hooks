## ADR-0010: backup dir 名に crypto random suffix を追加

**日時**: 2026-05-03T18:00:00+09:00
**タスク概要**: `runner.ts:603-608` の untracked file 用 backup dir 名 `${process.pid}_${Date.now()}` に `crypto.randomBytes(4).toString('hex')` を 1 要素追加して、PID 衝突 (worktree 横断で fork 後に PID 再利用が起きた直後の orphan dir 残存等) を完全に排除する。

### コンテキスト

現状: `.git/ts-git-hooks/backups/${pid}_${Date.now()}`

衝突条件は厳しい (同 PID は同時に hook 走らせない; 別 PID は PID 部分で必ず異なる) ので実害は出ていない。ただし PID wraparound (Linux で 32k 程度) 後に同 PID + 同 ms (Date.now の解像度) で旧 dir が残っていた場合、`mkdir -p` での合流が可能性として残る。`evacuateFiles → restoreFiles` の double-restore を起こさないために defense-in-depth。

### 決定内容

```ts
import { randomBytes } from "node:crypto";
// ...
evacuatedDir = join(
  ".git",
  "ts-git-hooks",
  "backups",
  `${process.pid}_${Date.now()}_${randomBytes(4).toString("hex")}`,
);
```

### 決定理由

- 衝突確率を実質ゼロに
- 1 行 (import 含めて 2 行) の純加算で副作用なし
- crypto は Node 標準なので追加 dep なし

### 却下した選択肢

| 選択肢 | 却下理由 |
|--------|----------|
| `crypto.randomUUID()` で全置換 | dir 名から PID/timestamp の人間可読性を失う (debug 時に有用) |
| `process.hrtime.bigint()` を使う | timestamp 解像度を上げるだけで PID 衝突は依然残る |
| 現状維持 | defensive 側で 1 行で消せる risk なら消しておく方が pareto 優位 |

### 影響範囲

test 影響:

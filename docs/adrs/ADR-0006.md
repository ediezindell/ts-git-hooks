## ADR-0006: install 時に dist/cli.js の絶対パスを hook script に埋め込み self-respawn を回避

**日時**: 2026-05-03T16:00:00+09:00
**タスク概要**: ADR-0005 で採用した B2 の実装。`install` 時に `process.argv[1]` から `dist/cli.js` の絶対パスを resolve し、生成する hook script の最優先分岐に `node --experimental-strip-types <abs-path> run <hook>` を書き込む。これにより `cli/index.ts` の self-respawn が install 経由では発火しなくなり、hook 1 回あたり 30〜100ms の Node start overhead が消える。

### コンテキスト

ADR-0005 で B2 を採用済み。本 ADR は実装方針・互換性・テスト戦略を確定する。

現状の hook script (`install.ts:31-45`):
```sh
if [ -x "./node_modules/.bin/ts-git-hooks" ]; then
  exec ./node_modules/.bin/ts-git-hooks run pre-commit
else
  exec npm exec ts-git-hooks run pre-commit
fi
```

`./node_modules/.bin/ts-git-hooks` 経路は `#!/usr/bin/env node` shim → `dist/cli.js` を起動 → flag 不在検出 → `--experimental-strip-types` 付きで子 Node を fork、という二段起動になる。

### 決定内容

1. **`install.ts` に `getDistCliPath()` helper を追加 (export)**:
   ```ts
   export function getDistCliPath(): string | null {
     const entry = process.argv[1];
     if (!entry || !entry.endsWith("cli.js")) return null;
     return entry;
   }
   ```
   - `process.argv[1]` は `npx ts-git-hooks install` 経由でも `node dist/cli.js install` 経由でも `dist/cli.js` の絶対パス
   - `cli.js` 終わりでない場合 (vitest 経由 / Yarn PnP の virtual path / 想定外の起動経路) は `null` を返し fallback
   - test では `process.argv[1]` がテストランナーになるので自然に `null` → 既存 fallback path が走り、現行の install.test.ts assertions と互換

2. **`getHookExecutionCommand` を改修して 3 段分岐に**:
   ```sh
   if [ -f 'ABS_PATH' ]; then
     exec node --experimental-strip-types 'ABS_PATH' run pre-commit
   elif [ -x "./node_modules/.bin/ts-git-hooks" ]; then
     exec ./node_modules/.bin/ts-git-hooks run pre-commit
   else
     exec npm exec ts-git-hooks run pre-commit
   fi
   ```
   - 絶対パス埋め込みは `shell-quote` の `quote([cliPath])` で escape (空白・single quote 含むパスを安全に)
   - 第 1 分岐失敗時の fallback として現行の `.bin` 経路と package manager 経路を残す
   - `cliPath === null` の場合は第 1 分岐を出力せず、現行 2 段 (= 既存 hook script) のまま

3. **`cli/index.ts:29-40` の `ensureRuntimeFlags` は削除しない**:
   - install 経由の hook 実行では第 1 分岐が `--experimental-strip-types` を最初から付けるので発火しない
   - 直叩き dev (`node dist/cli.js run pre-commit`) の safety net として残置

4. **`install.test.ts` を新規 case 追加**:
   - `process.argv[1]` を一時的に `/fake/dist/cli.js` に差し替えて optimized 経路を assert
   - `process.argv[1]` 未対応 path のときの fallback assert は既存通り
   - `getDistCliPath` の unit test (`endsWith` 判定と `null` 返却の境界)

### 決定理由

- `process.argv[1]` は Node の標準で entry script の絶対パスを返す。`require.resolve("ts-git-hooks/package.json")` 経由より shim/symlink の前提が薄く、Yarn PnP のような特殊環境では自然に null fallback できる
- 第 1 分岐が file 存在チェック付きなので、ユーザーが node_modules を消した・移動した場合は自動で fallback 経路に降りる。stale path の silent failure を避けられる
- `shell-quote` は既に依存に入っており追加 dep なし
- engines 据置 (ADR-0005 通り)

### 却下した選択肢

| 選択肢 | 却下理由 |
|--------|----------|
| `require.resolve("ts-git-hooks/package.json")` から path 算出 | 自パッケージ名解決が `npm link` / monorepo workspace で挙動ぶれ。`process.argv[1]` の方が薄くて安定 |
| `import.meta.url` + 相対 navigation | 鉤 chunk の location 前提 (`dist/chunks/install-X.js → ../cli.js`)。rolldown chunk 配置を変えた瞬間に壊れる |
| 第 1 分岐失敗時に hook 自体を失敗させる | install 後の node_modules 操作で hook が壊れる UX 劣化。fallback 維持 |
| `ensureRuntimeFlags` 削除 | dev safety net を失う。CLI を `node dist/cli.js` で直叩き確認できなくなる |
| Hook 内で `node -p "require.resolve(...)"` を使い動的に解決 | 余計な node start を 1 回挟み速度改善が相殺される |

### 影響範囲

変更されるファイル:
- `src/commands/install.ts` — `getDistCliPath` 追加、`getHookExecutionCommand` 改修、`shell-quote` import
- `src/commands/install.test.ts` — 新規 test ケース追加 (optimized path / null fallback / `getDistCliPath` 単体)

副次効果・互換性:
- 既存ユーザーで `.git/hooks/*` を再生成しない人は古い 2 段 script のまま。`npx ts-git-hooks install` を再実行することで新 script に切替
- 第 1 分岐 fail 時は現行と同等の挙動 (`.bin` 経路) に降りる
- Yarn PnP ユーザーは `process.argv[1]` が `.pnp.cjs` 経由の virtual path になり `cli.js` 終わりにならない可能性 → null fallback で現行同等
- node_modules を移動した場合: 第 1 分岐が file 存在 check で fail → `.bin` 経路へ自動降格

公開 API への影響:
- `getDistCliPath` を export するが `dist/index.d.ts` には載らない (commands は internal)
- 既存 export は不変

### Privacy / data flow check

- 保存する user 入力の種類: 絶対 path (`process.argv[1]` 経由)。これは local file system path で server に送られない
- 各 field が server に送られる必要があるか: No (hook script に書き込むのみ、`.git/hooks/` は per-clone)
- 採用する保存方式: `.git/hooks/<hook-name>` (既存の install 動作と同じ)
- 採用理由: hook script の役割上 local file 以外に置く理由なし

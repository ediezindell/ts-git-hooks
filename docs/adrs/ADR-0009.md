## ADR-0009: addFiles / getChangedFiles の file 引数を stdin (--pathspec-from-file) 経由に切替

**日時**: 2026-05-03T18:00:00+09:00
**タスク概要**: `git add` / `git diff --name-only` への file path 受け渡しを spawn 引数列から `--pathspec-from-file=- --pathspec-file-nul` 経由の stdin null-separated 入力に切替えて、OS の arg 上限 (macOS の ARG_MAX = 256KB) で hook が落ちる可能性を排除する。

### コンテキスト

`utils/git.ts:220-233` (`addFiles`) と `utils/git.ts:202-213` (`getChangedFiles`) は、files を `args.push("--", ...files)` で spawn 引数として渡している。Linux ARG_MAX は概ね 2MB で実害は少ないが、macOS は 256KB。1 commit で 1000+ ファイル × 平均 100 char path だと 100KB+ で上限が見える。monorepo + auto-format で全体に手が入るパターンで実際に起きる。

### 決定内容

1. **`utils/git.ts` に内部 helper `execGitWithStdin(args, stdin: string | Buffer): Promise<Buffer>` を追加**:
   - 既存 `execGitBuffer` と同骨格、`child.stdin.write(stdin); child.stdin.end()` を加えるだけ
   - DRY のため可能なら `execGitBuffer` と内部実装を共有

2. **`addFiles`**:
   ```ts
   export async function addFiles(files: string[], force = false): Promise<void> {
     if (files.length === 0) return;
     const args = ["add"];
     if (force) args.push("-f");
     args.push("--pathspec-from-file=-", "--pathspec-file-nul");
     await execGitWithStdin(args, files.join("\0"));
   }
   ```
   - `--` (option terminator) は `--pathspec-from-file` 使用時は不要。pathspec は file 経由で渡るので argument injection 余地が消える (security 副次効果)

3. **`getChangedFiles`** — `git diff` は `--pathspec-from-file` を受け付けないため、無パス引数で全件取得し JS 側で `Set.has` フィルタする:
   ```ts
   export async function getChangedFiles(files?: string[]): Promise<string[]> {
     if (files?.length === 0) return [];
     const allChanged = await execGitList([
       "diff", "--name-only", "--diff-filter=ACMR", "-z",
     ]);
     if (!files) return allChanged;
     const allowed = new Set(files);
     return allChanged.filter((f) => allowed.has(f));
   }
   ```
   - 同じ set semantics を保ちつつ argv を経由しない。`git diff` 自体は path arg なしでも全 worktree-vs-index 差分を返すので、結果は path-scoped 形式と同一

4. **README の "Requirements" 節を更新**: `Git 2.25+` (`git add` への `--pathspec-from-file` 導入バージョン) を明記

5. **`utils/git.test.ts` の既存 test を更新**: spawn 引数配列の expectation を新形式に揃え、`child.stdin.write` 呼出しのモック検証を追加

### 決定理由

- `addFiles`: stdin 経路は file 数に対して O(1) の arg list で済むため arg 上限から自由。`--pathspec-file-nul` で path に含まれる特殊文字も堅牢
- `getChangedFiles`: `git diff` には `--pathspec-from-file` が無いため、無パス引数で全件取得 + JS フィルタにすることで argv 経路を完全に消す。同じ set semantics を保つ
- git 2.25 (Jan 2020) は engines: Node 22.6 (Aug 2024) と整合するくらい古いので user 影響は実質なし

### 却下した選択肢

| 選択肢 | 却下理由 |
|--------|----------|
| OS 別に arg 列の長さを実測しバッチ分割 | 実装複雑化、batch 跨ぎで semantic 変化 (例: `git add` 部分成功) を扱う必要 |
| 現状維持 + README に「大量ファイル時は手動分割」と注記 | UX 悪化、defensive な library の役割を放棄 |
| `xargs` 経由に shell-out | `shell: true` 化、依存の方向が逆 (現状 spawn だけで完結している)、Windows 互換性 |
| feature detection (git version) で fallback | 二系統保守の cost。git 2.25 は十分古い |
| `getChangedFiles` でも `--pathspec-from-file` を使う | 実装中に `git diff` が当該 option を受け付けないことを発見し却下。代替として無パス + JS `Set.has` フィルタを採用 |

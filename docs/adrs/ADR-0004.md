## ADR-0004: 存在チェックを micromatch 後ろへ移し lstat 化

**日時**: 2026-05-03T15:00:00+09:00
**タスク概要**: 速度改善 Tier 1。staged files の存在チェック (`stat`) を `getStagedFiles()` 直後の全件適用から `resolveScriptsToRun` 内へ移し、glob hook では mm 結果の subset に絞って `lstat` で行うよう変える。

### コンテキスト

`runHook` は `runner.ts:685-696` で `Promise.all(stagedFiles.map(stat))` により staged 全 N 件に対して existence check を行っている。これは:

- pre-commit で staged files が多い (例: monorepo で 500 files commit) かつ glob match が小さい (`*.ts` で 5 files) ケースで O(N) の syscall を浪費する
- `restoreFiles` (`utils/git.ts:354`) は symlink を follow しないよう `lstat` を使うが、ここは `stat` で symlink を follow する。symlink target が消えていれば false negative で除外される非対称が存在

### 決定内容

1. **`runHook` 内の事前 stat ループを削除** (`runner.ts:685-696`)
2. **`resolveScriptsToRun` に内製 helper `filterExisting(files)` を追加**: 並列 `lstat`、catch で除外
3. **glob hook 経路: `mm()` 直後に `matchedFiles = await filterExisting(matchedFiles)`** を挿入 (`runner.ts:256` 付近)
   - 単一 pattern 経路でも複数 pattern 経路でも、`matchedFiles` を絞ってから downstream の `addCommandBatch` / `groupPatternsByCommand` に渡す
4. **simple hook 経路: 関数冒頭で `stagedFiles = await filterExisting(stagedFiles)`** を適用
   - simple+tuple のとき staged files が argsFn に渡る semantics を保つため、現状 N → N (win なし) でも symlink 整合のため lstat 化は維持
5. **`runner.test.ts:36-42` の `vi.mock("node:fs/promises")` に `lstat: vi.fn().mockResolvedValue({})` を追加**。`stat` mock は本コードからの参照が消えるが、他の経路 (e.g. `node:fs/promises` 由来の他関数) で必要なら残す

### 決定理由

- glob hook (= pre-commit) で M << N の場合に N-M 個の syscall を削減できる。pre-commit が最多用途なので体感に効く
- `lstat` 統一で symlink semantics が `restoreFiles` と一致し、broken symlink の除外/包含が両者で揃う
- API contract (`resolveScriptsToRun` の I/O signature) は不変。`matchedFiles` の semantic も従来踏襲 (mm 結果 ∩ 存在ファイル)

### 却下した選択肢

| 選択肢 | 却下理由 |
|--------|----------|
| 事前 stat フィルタを完全削除 (検証なし) | staged 後に worktree から消えた file が linter に渡って失敗するエッジケースが復活する。git の `--diff-filter=ACMR` は index しか見ないため worktree-only delete を検出できない |
| `lstat` ではなく `access(F_OK)` | 速度差は誤差。`access` は symlink target の存在を見るので `stat` と同じ非対称が残り利点なし |
| simple+tuple 経路では filter を行わない | 「現状 existingStagedFiles で除外されていた non-existent」が argsFn に流れ込む挙動変化が発生。後方互換のため filter は維持 |
| `runHook` 側に `filterExisting` を残し `resolveScriptsToRun` には触らない | glob hook で N→M に絞れる win が消える (本提案の主目的) |

### 影響範囲

変更されるファイル:
- `src/core/runner.ts` — `runHook` から stat ループ削除、`resolveScriptsToRun` 内に lstat filter 追加
- `src/core/runner.test.ts` — `node:fs/promises` mock に `lstat` 追加

公開 API (`d.ts`) への影響:
- `resolveScriptsToRun` signature 不変
- `matchedFiles` semantic 不変

挙動変化:
- broken symlink の扱い: 旧 `stat` → 除外、新 `lstat` → 含む。glob match の対象になりうる broken symlink を staged している場合、linter に渡る。実用上 broken symlink を commit するケースは稀かつ linter が fail するのは正しい挙動

## ADR-0005: self-respawn 撤廃の実装方針として install 時の絶対パス埋め込み (B2) を採用

**日時**: 2026-05-03T15:00:00+09:00
**タスク概要**: 速度改善 Tier 2 として CLI の `--experimental-strip-types` self-respawn を撤廃する。実装に先立ち、approach (B1: `NODE_OPTIONS` 経由 vs B2: install 時に bin の絶対パスを resolve して `node --experimental-strip-types <abs-path>` 形式で hook script に書き込む) のどちらを採るかを記録する。本 ADR は decision のみで実装は別 ADR。

### コンテキスト

現状 `cli/index.ts:29-40` の `ensureRuntimeFlags()` は flag 不在を検出すると子 Node を fork して `--experimental-strip-types` 付きで再起動している。hook 1 回あたり 30〜100ms の Node start overhead。

撤廃の選択肢:

- **B1**: hook script で `NODE_OPTIONS="--experimental-strip-types" exec ./node_modules/.bin/ts-git-hooks ...` を設定
  - Node 22.7+ で `--experimental-strip-types` が `NODE_OPTIONS` allowlist に追加された (以前は reject)
  - 現 `engines: ">=22.6.0"` だと 22.6.x ユーザーで NODE_OPTIONS allowlist が異なる → engines bump 必要
- **B2**: install 時に `require.resolve('ts-git-hooks/dist/cli.js')` で bin 実体パスを取得し、hook script に `node --experimental-strip-types <abs-path> run <hook>` を直接書き込む
  - Node version 別挙動なし (engines 据置)
  - 絶対 path が `.git/hooks/` に書かれるが、`.git/` は per-clone なので発生し得る paths は各 dev 環境固有で問題なし

### 決定内容

**B2 を採用**。

将来の実装 ADR で記す予定の方針:

1. `install.ts:31-46` の `getHookExecutionCommand` を改修
2. install 時に `require.resolve('ts-git-hooks/dist/cli.js')` で絶対パスを取得
3. hook script は `if [ -x ... ]` 分岐を保ったまま、ローカル resolve できたケースだけ最適化経路に乗せる:
   ```sh
   exec node --experimental-strip-types /abs/path/to/dist/cli.js run pre-commit
   ```
   resolve 失敗時は現行の `${packageManager} ts-git-hooks run ...` を fallback として維持
4. `cli/index.ts:29-40` の `ensureRuntimeFlags` は **削除しない**: dev 用途 (`node dist/cli.js` 直叩き) の safety net として残す。install 経路では発火しないので overhead は消える

### 決定理由

- engines 据置で既存 user に影響を出さない
- Node version 別の挙動分岐が不要で予測可能
- 将来 NODE_OPTIONS allowlist 更新で B1 が魅力的になっても、B2 採用は B1 への移行余地を恒久排除しない (移行は別 ADR で扱える)

### 却下した選択肢

| 選択肢 | 却下理由 |
|--------|----------|
| B1 (NODE_OPTIONS) | engines bump が user 都合に対して負担。22.6 ↔ 22.7 の差は小さいが、本 project が breaking しない方針なら避ける |
| B3 (B1 + B2 fallback) | 実装複雑化 (bin script で Node version 検出 → 分岐)。利得は B2 単独に対して ergonomics 微増のみ |
| `ensureRuntimeFlags` を削除して install 時に flag 必須化、CLI 単体実行を不可とする | dev 環境 (`node dist/cli.js` で動作確認) を壊す |
| 現状維持 | hook 1 回あたり 30〜100ms の overhead を放置。Tier 2 の主目的を放棄 |
